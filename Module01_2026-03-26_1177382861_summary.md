## Key Concepts Explained

### Thursday doubt-clearing sessions
- **What it is:** Weekly Thursday classes dedicated to student questions, not scheduled curriculum. Off-curriculum topics (e.g., advanced ML, DevOps, AI Ops) can be discussed if relevant; later, project work may spill into Thursdays if regular classes run out of time.
- **Why it matters:** Use these sessions for doubts, career questions, or extra topics—not expecting new syllabus content by default.
- **Instructor emphasized:** Post doubts in `#doubts` during the week instead of waiting for Thursday; questions from the channel may already be answered when you watch the recording. Attendance on doubt days typically drops over time—don’t rely on Thursdays alone.

### Bootcamp schedule and break week
- **What it is:** Regular teaching is **Saturday and Sunday**. After **week 2** (the upcoming weekend), there is a **two-week break week** with **assignments** to practice Python, especially for non-technical backgrounds.
- **Why it matters:** Python practice during break is intentional—“boring but necessary” before moving on.
- **Instructor emphasized:** You may use AI for help on assignments; tag the instructor on `#doubts` for questions. Optional study groups of **3–4 people** (not 8–10); instructor can help form groups via breakout rooms but won’t force it.

### Discord and platform support
- **What it is:** **`#doubts`** for technical questions (tag instructor). **`#general-chat`** for Codecademy platform issues (tag **Nazia** or **Aparna**). General discussion uses general chat, not the doubts channel.
- **Why it matters:** Correct channel = faster help; live Q&A is for in-class only.
- **Caveats:** Discord link is on the dashboard and in chat; join may require display name and date of birth. Some students had join/access issues handled offline.

### Dashboard “Join session”
- **What it is:** From bootcamp **View details**, **Join session** at the top activates when class is live (about **five minutes before** start).
- **Why it matters:** If the button stays gray, **refresh the page** within that window.

### Client delivery of AI (consulting example)
- **What it is:** Delivery model depends on project type. Example: bank client (e.g., HSBC) hires a consulting firm (e.g., Palo IT); AI engineer builds a **generative AI chatbot** after discovery on security, **private vs. public data**, **data governance**, preferred **cloud**, and **LLM** choices.
- **Why it matters:** Enterprise work is preference-driven; the engineer proposes options (e.g., AWS) with pros/cons before build.
- **Example deliverable:** Python app on **AWS EC2**, client files in **S3**, LLMs via **AWS Bedrock**, code in client’s **GitHub**—then hand off code and deployment for the client to operate. Many students won’t know all jargon yet; that’s expected early in the bootcamp.

### Subscription / token-based tools vs. client handoff
- **What it is:** Client-facing solutions are often deployed on **client-owned infrastructure** (cloud, repos), not “ship a ChatGPT subscription.”
- **Why it matters:** Addresses concern that SaaS APIs tie deliverables to vendor subscriptions—the consulting pattern is deployable assets on the client’s stack.

### Technical vs. non-technical AI careers
- **What it is:** **Non-technical paths** include agentic AI roles using **no-code** tools, **business analyst** (domain + AI + communication), **data analytics / AI manager**, **product / project manager** (AI literacy without coding). **UX designer** was mentioned; **UI developers with AI** are technical but in demand. **Technical paths** include AI engineering, ML/AI, data science, data analyst, agentic AI engineering/developer, GenAI agent/engineering roles—**all need strong Python** in the instructor’s view.
- **Why it matters:** Bootcamp value extends even without a full career switch—AI is becoming a **secondary or tertiary skill** across roles (e.g., mandatory prompt/agentic training at large firms).
- **Instructor emphasized:** Non-technical folks should still try Python; he moved from Java to Python in ~10 days with prior programming experience. Research roles beyond what was listed.

### Vibe coding and code assistants
- **What it is:** Using AI (e.g., ChatGPT, Gemini, **Claude Code**-class tools) to generate code—“vibe coding.” Instructor plans to use AI when building **agentic AI projects** in the bootcamp.
- **Why it matters:** Practical for agentic apps **after** fundamentals; risks include **hallucinations** and lack of **factual correctness**.
- **Instructor emphasized:** Learn Python without shortcuts first (no Colab **hash-comment auto-fill** while learning—type code yourself). **Fact-check** before using AI-generated code; don’t blindly copy-paste. Don’t rely on ChatGPT for what’s “common” in curriculum (e.g., list comprehensions).

### Data science vs. AI
- **What it is:** Treated as closely related: AI → ML → deep learning; all are **data-centric**; studying data in that space is **data science** (analogous to computer science studying computers).
- **Why it matters:** “Python for Data Science” skill path aligns with the bootcamp path—not a separate unrelated track.

### Python topics still coming
- **What it is:** **Two more weekend classes** to wrap Python; **list comprehensions** in the **next** class. Day-to-day work will lean on **NumPy, Pandas**, and similar libraries more than raw lists/tuples/dicts/sets.
- **Why it matters:** List comprehensions are **not** framed as commonly used in this bootcamp’s downstream work—you can deprioritize them vs. libraries.

### Placement assistance
- **What it is:** **Career guidance**, **resume prep**, and **mock interviews**—not a **job guarantee**.
- **Why it matters:** Sets expectations for post-bootcamp support.

### Live bootcamp pacing
- **What it is:** Live format enables interaction and live Q&A; drawback is **one pace** for a mixed-ability cohort.
- **Why it matters:** Students ahead in Python can **DM instructor on Discord** to discuss jumping ahead on curriculum.
